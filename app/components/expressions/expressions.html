Expressions

Angular helps bridge the gap between your html, css, and js

A cool feature of angular is the ability to use Expressions in our html

Example: 2+2 = {{ 2 + 2 }}

Angular parses your html and looks specifically for the double curlies, handlebars, or mustache brackets. When angular comes across a set of these brackets it essentially says this

Pause HTML {{ Evaluate JavaScript code: }} Restart HTML

Writing a full scale application with techniques like above quickly become cumbersome and difficult to read. Thanks to angular HTML Templating becomes a lot easier as you will see as you continue through. Look at the example below to see the angular approach

Angular Approach

		
<div class="player-card">
	<button class="btn btn-xs btn-danger remove">Remove</button>
	<div>
		<img ng-src="player.photo" />
	</div>
	<div>
		<span>{{player.name}}</span>
	</div>
	<div>
		<span>{{player.position}}</span>
	</div>
	<div>
		<span>{{player.number}}</span>
	</div>
</div>

What is nice about this approach is you don't have to leave your markup while you are trying to figure out how you want something to look. Also you never have to write using string concatination and keep track of all the '""' + '""'
